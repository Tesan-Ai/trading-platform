4. Walk-forward validation

walk_forward_runner.py is implemented.

It:
- Runs fold-based walk-forward validation.
- Defaults to 2025-09-03 through 2026-06-03.
- Defaults to 1-month folds.
- Runs replay/backtesting per fold.
- Aggregates trades and reports.
- Calculates metrics.
- Applies validation gates.
- Builds strategy scorecards.
- Records experiments to research_results/experiments.

Example command from PLATFORM_ARCHITECTURE.md:

.venv/bin/python walk_forward_runner.py \
  --strategy opening_range_vwap_momentum_v1 \
  --start-date 2025-09-03 \
  --end-date 2026-06-03 \
  --orvwap-quality-profile

The OR/VWAP quality profile:
- Relaxes filters.
- Uses MSFT/META only by excluding NVDA, TSLA, AAPL, AMZN, AMD.

5. Validation gates

validation/performance_gate.py is implemented.

Backtest gate checks:
- Minimum closed trades.
- Profit factor.
- Expectancy.
- Max drawdown.
- Win rate.

Default backtest thresholds:
- Minimum closed trades: 30
- Minimum profit factor: 1.15
- Minimum expectancy: above 0.0
- Maximum drawdown: 8%
- Minimum win rate: 40%

Gate statuses:
- Failed backtest = RESEARCH_ONLY
- Passed backtest = PAPER_CANDIDATE
- Passed paper/live gate = LIVE_CANDIDATE

11. Alpaca paper broker

brokers/alpaca_paper.py is implemented.

It:
- Uses ALPACA_API_KEY and ALPACA_SECRET_KEY.
- Connects to Alpaca TradingClient with paper=True.
- Fetches current broker positions.
- Saves positions to Supabase if enabled.
- Submits market orders.
- Closes positions.
- Saves order records to Supabase if enabled.

This is a paper broker implementation, not live trading.

12. Broker reconciliation

brokers/reconciliation.py is implemented.

It compares:
- Local CSV/open positions
- Broker positions

It detects:
- Positions missing locally.
- Positions missing at broker.
- Quantity mismatches.

If reconciliation fails:
- It returns ok=False.
- It records a BROKER_RECONCILIATION_FAILED risk event to Supabase if enabled.
- paper_trader.py blocks new entries when broker reconciliation fails.

There is also:
reconcile_runner.py

This is a standalone runner for reconciliation.

13. Paper trading cycle

paper_trader.py is implemented as the live-ish/paper execution engine.

It:
- Fetches recent market data from Alpaca.
- Builds features.
- Resolves market regime.
- Estimates current equity.
- Applies RiskGate.
- Loads local open positions.
- If broker is enabled, reconciles local and broker positions.
- Evaluates exits.
- Places sell/close actions through broker if configured.
- Logs sells.
- Checks entry window.
- Checks risk gate.
- Checks market regime.
- Evaluates entries.
- Places buy orders through broker if configured.
- Updates local CSV positions.

TRADING_MODE=SIGNAL_ONLY means no broker.
BROKER_PROVIDER=csv keeps old local CSV behavior.
BROKER_PROVIDER=alpaca_paper connects to Alpaca paper.
